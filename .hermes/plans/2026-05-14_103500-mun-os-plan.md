# MUN OS 开发计划

## 技术栈
- **后端**: Python + FastAPI + SQLAlchemy + SQLite
- **前端**: Vue 3 + Element Plus + Vue Router + Pinia
- **认证**: JWT Token
- **通信**: REST API + WebSocket（实时会议功能）

## 数据库设计

### 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增ID |
| username | VARCHAR(50) | 用户名（唯一） |
| password_hash | VARCHAR(128) | 密码哈希 |
| role | VARCHAR(20) | 角色：admin/staff/delegate |
| created_by | INTEGER FK | 创建者ID |
| committee_id | INTEGER FK | 所属委员会ID（学团用） |
| delegation_id | INTEGER FK | 所属代表团ID（代表用） |
| is_leader | BOOLEAN | 是否阁首（代表用） |
| created_at | DATETIME | 创建时间 |

### 委员会表 (committees)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增ID |
| name | VARCHAR(100) | 委员会名称 |
| features | JSON | 可用功能列表 |
| created_at | DATETIME | 创建时间 |

### 代表团表 (delegations)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增ID |
| name | VARCHAR(100) | 代表团名称 |
| committee_id | INTEGER FK | 所属委员会ID |
| created_at | DATETIME | 创建时间 |

### 议程表 (agenda_items)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增ID |
| committee_id | INTEGER FK | 所属委员会ID |
| title | VARCHAR(200) | 议程标题 |
| level | INTEGER | 层级（分级议程） |
| order | INTEGER | 排序序号 |
| is_active | BOOLEAN | 是否当前议程 |
| created_at | DATETIME | 创建时间 |

### 动议表 (motions)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增ID |
| committee_id | INTEGER FK | 所属委员会ID |
| type | VARCHAR(30) | 类型：moderated_caucus/unmoderated_caucus/speakers_list |

| unit_duration | INTEGER | 单位时长（秒） |
| total_duration | INTEGER | 总时长（秒） |
| status | VARCHAR(20) | 状态：pending/active/paused/ended |
| created_at | DATETIME | 创建时间 |

### 发言名单表 (speakers_list)
| 字段 | 类型 | 说明 |

### 指令表 (directives)

### 文件表 (documents)

## 开发阶段

### 阶段一：基础架构（当前）
1. 初始化后端项目（FastAPI + SQLAlchemy + SQLite）
2. 初始化前端项目（Vue 3 + Vite + Element Plus）
3. 实现用户认证（登录/JWT）
4. 管理员端：创建/删除学团账密、创建/删除委员会、分配学团到委员会

### 阶段二：会前准备
1. 学团端：创建/删除代表账密、指定阁首
2. 学团端：创建/删除代表团、分配代表
3. 学团端：创建分级议程单
4. 管理员端：选择委员会可用功能

### 阶段三：会中核心
1. 学团端：点名功能
2. 学团端：会议主持（动议管理、发言计时器、发言名单）
3. 代表端：提交指令/文件
4. 学团端：接收指令/文件
5. 学团端：发布局势更新/文件
6. 代表端：接收局势更新/文件

### 阶段四：高级功能
1. WebSocket 实时通信（会议状态同步）
2. 存档/恢复会议状态
3. 发言记录统计
